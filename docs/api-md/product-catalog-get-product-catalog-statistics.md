# Get Product catalog Statistics

## Overview
Retrieve comprehensive statistics for an entire product catalog including summary metrics, VAT breakdown, and top-selling products within an optional date range.

## Prerequisites
- Merchant account setup.
- Valid `catalogId` is required.
- Authentication via API Key, API Secret and Merchant ID.

## Request

### HTTP Method and URL
```
GET /catalog/:catalogId/statistics
```

### Headers
| Header | Value | Required |
|---|---|---|
| Content-Type | application/json | Yes |
| API-KEY | YOUR_API_KEY | Yes |
| API-SECRET | YOUR_API_SECRET | Yes |
| MERCHANT-ID | YOUR_MERCHANT_ID | Yes |

### Query Parameters
| Parameter | Type | Required | Description |
|---|---|---|---|
| startDate | string | No | Start date of the statistics period in YYYY-MM-DD format. |
| endDate | string | No | End date of the statistics period in YYYY-MM-DD format. |

### Request Body Parameters
N/A

### Request Example
```json
{
  "headers": {
    "Content-Type": "application/json",
    "API-KEY": "YOUR_API_KEY",
    "API-SECRET": "YOUR_API_SECRET",
    "MERCHANT-ID": "YOUR_MERCHANT_ID"
  }
}
```

## Response

### Response Parameters
| Parameter | Type | Description |
|---|---|---|
| status | string | Status of the request, indicated as either 'SUCCESS' or 'ERROR'. |
| data | object | Response data containing catalog statistics. |
| data.productcatalogId | string | Unique identifier of the product catalog. |
| data.productcatalogName | string | Name of the product catalog. |
| data.query | object | The date range used for filtering statistics. |

| data.summary.totalProducts | number | Total number of products in the catalog. |
| data.summary.totalVariants | number | Total number of variants across all products. |

| data.summary.byCurrency[].currency | string | Currency code. |

| data.topSellingProducts[].productId | string | Unique identifier of the product. |
| data.topSellingProducts[].productName | string | Name of the product. |

| data.topSellingProducts[].currencyBreakdown[].totalRevenue | number | Total revenue in this currency. |
| message | string | A message that describes the status of the request. |

## Error Responses

### Possible Errors

#### 400 - Bad Request
```json
{
  "status": "ERROR",
  "data": null,
  "message": "Invalid date format. Date should be in YYYY-MM-DD format."
}
```
**Description:** The request was malformed or contained invalid parameters, such as an incorrect date format.  Ensure the `startDate` and `endDate` parameters are in the correct YYYY-MM-DD format.

#### 401 - Unauthorized
```json
{
  "status": "ERROR",
  "data": null,
  "message": "Unauthorized: Invalid API key."
}
```
**Description:** The API key provided in the request headers is invalid or missing. Verify the API key is correct and included in the request.

#### 403 - Forbidden
```json
{
  "status": "ERROR",
  "data": null,
  "message": "Forbidden: Insufficient permissions."
}
```
**Description:** The merchant does not have the necessary permissions to access the resource. Check if the merchant ID is correct and if the user associated with the API key has the required roles.

#### 404 - Not Found
```json
{
  "status": "ERROR",
  "data": null,
  "message": "Product catalog not found."
}
```
**Description:** The specified `catalogId` does not exist. Verify the `catalogId` is correct and active.

#### 500 - Internal Server Error
```json
{
  "status": "ERROR",
  "data": null,
  "message": "An unexpected error occurred on the server."
}
```
**Description:** An unexpected error occurred on the server. This is usually a temporary issue. Retry the request after a few minutes. If the problem persists, contact support.

## Notes
- Ensure that the `catalogId` used in the request path is a valid and existing catalog ID.
- The `startDate` and `endDate` query parameters are optional. If not provided, statistics for all available data will be returned.
- Dates should be provided in the format `YYYY-MM-DD`.
