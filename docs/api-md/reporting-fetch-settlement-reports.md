# Fetch Settlement Reports

## Overview
Retrieves the list of settlement reports for a merchant. Use this API to get a summary of the merchant’s settled transactions within a specific time frame, which can be daily or monthly depending on the report type opted for the merchant.  This API supports pagination.

## Prerequisites
- A valid Partner ID and Merchant ID are required.
- The Merchant must have existing settlement reports.
- Authentication is required using API Key and API Secret.

## Request

### HTTP Method and URL
```
GET /partners/:partnerId/merchants/:merchantId/reports
```

### Headers
| Header | Value | Required |
|---|---|---|
| Content-Type | application/json | Yes |
| API-KEY | YOUR_API_KEY | Yes |
| API-SECRET | YOUR_API_SECRET | Yes |
| MERCHANT-ID | YOUR_MERCHANT_ID | Yes |

### Query Parameters
N/A

### Request Body Parameters
N/A

### Request Example
```json
{
  "headers": {
    "Content-Type": "application/json",
    "API-KEY": "YOUR_API_KEY",
    "API-SECRET": "YOUR_API_SECRET",
    "MERCHANT-ID": "YOUR_MERCHANT_ID"
  }
}
```

## Response

### Response Parameters
| Parameter | Type | Description |
|---|---|---|
| status | string | Status of the request, indicated as either 'SUCCESS' or 'ERROR'. |
| data | array | Response data |
| data[].payoutId | String | The payout ID used to identify a specific payout. |
| data[].merchantId | String | The merchant ID of the merchant for which the settlement report is generated. |
| data[].transactionStartDate | String | The date of the first transaction in the generated settlement report, in YYYY-MM-DD format. |
| data[].transactionEndDate | String | The date of the last transaction in the generated settlement report, in YYYY-MM-DD format. |
| data[].settlementDate | String | Date of the payout, in YYYY-MM-DD format. |
| data[].reportType | String | Specifies the type of settlement report opted for the merchant. It can be either 'MONTHLY' or 'DAILY’. |
| data[].url | String | URL from which the particular settlement report can be viewed. |
| data[].totalSale | Number | The total amount of sales generated by the merchant during the report period. |
| data[].totalRefund | Number | The total amount refunded by the merchant during the specified time period. |
| data[].fee | Number | The total fee collected from transactions during the settlement. |
| data[].payout | Number | The final amount paid out to the merchant after all deductions. |
| message | string | A message that describes the status of the request. |

## Error Responses

### Possible Errors
There is no error JSON example given, so these are examples based on context:

#### 400 - Bad Request
```json
{
  "status": "ERROR",
  "message": "Invalid Merchant ID"
}
```
**Description:**  The provided Merchant ID is not valid.  Check the Merchant ID and try again.

#### 401 - Unauthorized
```json
{
  "status": "ERROR",
  "message": "Invalid API Key"
}
```
**Description:**  The provided API Key is incorrect or missing. Verify your API Key and Secret.

#### 404 - Not Found
```json
{
  "status": "ERROR",
  "message": "Reports not found for specified merchant"
}
```
**Description:** No settlement reports exist for the given Merchant ID and Partner ID combination.

## Notes
- The `url` field in the response provides a direct link to download the settlement report in PDF format.
- The report type can be either DAILY or MONTHLY depending on the merchant's configuration.
- Ensure that the `transactionStartDate` and `transactionEndDate` are in the correct format (YYYY-MM-DD).
- This endpoint supports pagination.  Check for standard pagination parameters if needed (e.g., `page`, `pageSize`). The JSON did not give specific details of these.
