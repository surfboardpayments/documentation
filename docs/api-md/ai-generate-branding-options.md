# Generate Branding Options

## Overview
Generate AI-powered branding options and color schemes based on a website URL. Use this API to automatically create cohesive branding themes that match your business identity and enhance your merchant portal appearance.

## Prerequisites
- An active account with API access.
- API key, API secret, and merchant ID.
- The target website must be publicly accessible.

## Request

### HTTP Method and URL
```
POST /ai/branding
```

### Headers
| Header | Value | Required |
|--------|-------|----------|
| Content-Type | application/json | Yes |
| API-KEY | YOUR_API_KEY | Yes |
| API-SECRET | YOUR_API_SECRET | Yes |
| MERCHANT-ID | YOUR_MERCHANT_ID | Yes |

### Request Body Parameters
| Parameter | Type | Required | Description |
|---|---|---|---|
| url | string | Yes | Website URL to analyze for branding inspiration and color scheme generation. |

### Request Example
```json
{
	"url": "https://example-company.com"
}
```

## Response

### Response Parameters
| Parameter | Type | Description |
|---|---|---|
| status | string | Status of the request, indicated as either 'SUCCESS' or 'ERROR'. |
| data | object | Response data containing generated branding options. |
| data.options | array | Array of branding option sets with metadata. |
| data.options[].brandingOptions | array | Array of branding configuration options. |
| data.options[].brandingOptions[].backgroundColor | string | Hex color code for the background color. |
| data.options[].brandingOptions[].brandColor | string | Hex color code for the primary brand color. |
| data.options[].brandingOptions[].accentColor | string | Hex color code for accent elements. |
| data.options[].brandingOptions[].rectShape | string | Shape style for rectangular elements (e.g., 'rounded', 'square'). |
| data.options[].brandingOptions[].fontType | string | Font family type for text elements (e.g., 'mono', 'serif', 'sans-serif'). |
| data.options[].brandingOptions[].logoUrl | string | URL of the generated or suggested logo image. |
| data.options[].brandingOptions[].iconUrl | string | URL of the generated or suggested icon image. |
| data.options[].brandingOptions[].footerColor | string | Hex color code for footer elements. |
| data.options[].metadata | object | Metadata about the AI generation process. |
| data.options[].metadata.inputTokens | number | Number of input tokens used in the AI generation. |
| data.options[].metadata.outputTokens | number | Number of output tokens generated by the AI. |
| data.options[].metadata.outputType | string | Type of output generated (e.g., 'text', 'image'). |
| message | string | A message that describes the status of the request. |

### Success Response Example

{
	"status": "SUCCESS",
	"data": {
		"options": [
			{
				"brandingOptions": [
					{

## Error Responses

### Possible Errors

#### 400 - Bad Request
```json
{
  "status": "ERROR",
  "message": "Invalid URL provided."
}
```
**Description:** The provided URL is malformed or not a valid website address. Ensure the URL is correctly formatted and accessible.

#### 500 - Internal Server Error
```json
{
  "status": "ERROR",
  "message": "AI generation failed due to an internal server error."
}
```
**Description:** The AI service encountered an unexpected error during branding generation. Retry the request later. If the problem persists, contact support.

#### 401 - Unauthorized
```json
{
  "status": "ERROR",
  "message": "Invalid API key, secret or merchant ID."
}
```
**Description:** The API key, API secret, or merchant ID provided in the request headers are invalid or missing. Ensure that you have provided valid credentials in the request headers.

## Notes
- The AI algorithm will analyze the provided website's content, design, and color scheme to generate relevant branding options.
- The quality of the generated branding depends on the complexity and richness of the website's visual elements.
-  LogoUrl and IconUrl might be empty strings when no appropriate image are found from the provided Url.
